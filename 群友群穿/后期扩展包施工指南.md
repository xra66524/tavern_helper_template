## 一、机构子 Schema：制度详情嵌套

### 1.1 设计思路

把原"后期扩展包"的 14 个低频制度字段，从平铺 `extend` 改为 `机构Schema` 的一个**嵌套子对象** `制度详情`。该子 Schema 随机构一起存在：

- AI 在机构 `insert`/`rename`/改组时一并填写 `制度详情`
- 正常轮次保留原值（与机构高频字段同样的"低频不动"原则）
- **无需启用开关、无锁定、无 schema 切换**——建政后 AI 自然按剧情为机构补 `制度详情`
- 前端不新增 Tab，而是在机构节点点击时**弹出该机构的制度详情面板**（见第七章）

原 `制度备注` 自由文本字段保留以兼容旧数据；扩展后 AI 优先填 `制度详情` 子对象，`制度备注` 退化为自由补充说明，非必填。

### 1.2 机构 Schema：追加 `制度详情` 子 Schema

在 `mvu_schema.ts` 中，为现有 `机构Schema` 追加一个可选嵌套字段 `制度详情`，不破坏原有 11 个高频字段：

```ts
// ─── 机构制度详情子 Schema（嵌套于 机构Schema）────────────────────────────
// 14 个低频制度字段合并为一个子对象，随机构一起存在。
// 所有字段均为 z.string()——不硬约束 enum，AI 可按剧情扩展。
// 参考列表仅作 AI 填写参考，见 constants.ts 或 变量更新规则.yaml。
// AI 仅在机构 insert/rename/改组时填写，正常轮次保留原值。

const 机构制度详情Schema = z.object({
  // ── 党政二元结构与归属 ──
  所属体系: z.string().describe('党/政/军/群团/企事业。党政二元结构核心：党体系设书记+副书记+常委（民主集中制），政体系设行政首长（首长负责制）+党组书记（交叉任职），军体系保留双首长（政委+军事主官）'),
  机构性质: z.string().describe('常设机构/议事协调机构/临时机构/派出机构。议事协调机构编制虚、人员兼职，须填 依托部门'),
  级别: z.string().describe('机构层级，常见取值：最高决策/中枢军事/中枢行政/中枢办公/下属部门/基层单位/临时机构/司法监察/地方政权/国营企业/野战部队/民兵/秘密机构/意识形态机构，可按剧情扩展'),
  职务体系: z.string().describe('一把手称呼体系，如 总书记-常委/主席-副主席/主任-副主任/连长-排长'),

  // ── 条块关系 ──
  领导体制: z.string().describe('垂直管理/属地管理/双重领导/业务指导。双重领导=既受条条业务领导又受块块属地领导'),
  上级业务主管: z.string().describe('条条上的业务主管机构名，如 装备室的上级业务主管为中央军委；无则填 "无"'),

  // ── 决策与议事 ──
  决策机制: z.string().describe('常委会集体决策/一把手拍板/扩大会议议决/请示上级'),
  议事规则: z.string().describe('民主集中制/首长负责制/委员分工负责'),

  // ── 人事与编制 ──
  任命权限: z.string().describe('该机构负责人的任命主体，如 "政治局常委会任命"/"军委任命"'),
  职级: z.string().describe('主官职级，如 正国级/副国级/正部级/正厅级/正处级/正科级/科员'),
  编制数: z.coerce.number().describe('核定编制名额，是硬约束'),

  // ── 议事协调机构特性 ──
  依托部门: z.string().describe('议事协调机构的办事机构设在哪个常设部门；常设机构填 "无"'),
  组长级别: z.string().describe('议事协调机构组长的职级，必须高于或等于所协调机构最高级别；常设机构填 "无"'),

  // ── 信息流转与保密 ──
  请示报告要求: z.string().describe('该机构需向谁请示/报告哪些事项，如 "重大行动须请示政治局常委会，每周报告工作"'),
  保密等级: z.string().describe('公开/内部/机密/绝密。基地车相关机构通常为机密或绝密'),
});

// ─── 接入现有 机构Schema ────────────────────────────────
// 在原有 11 个高频字段基础上追加一个可选嵌套字段 制度详情。
// 初期（混乱期/临时领导小组期）机构可省略 制度详情；建政后 AI 按剧情补。
const 机构Schema = z.object({
  // ...原有 11 个高频字段保持不变（类型/一把手/二把手/.../上级机构/制度备注）
  制度详情: 机构制度详情Schema.optional().describe('机构制度深度建模子对象。建政后按需补；初期可省略。AI 在机构 insert/rename/改组时填写，正常轮次保留原值'),
});

// 注：原有 制度备注 字段保留（兼容旧数据）。扩展后 AI 优先填 制度详情 子对象，
//      制度备注 退化为自由补充说明，非必填。
```

### 1.3 兜底：机构 `insert` 时自动补空 `制度详情`

无需启用开关。兜底脚本在每轮 `VARIABLE_UPDATE_ENDED` 时，对已有机构按需补空 `制度详情` 默认值，AI 在后续轮次按剧情修正：

```ts
eventOn(Mvu.events.VARIABLE_UPDATE_ENDED, (variables) => {
  const 机构表 = _.get(variables, 'stat_data.组织结构.机构', {});
  for (const [机构名, 机构] of Object.entries(机构表)) {
    // 仅对已建政、有实有人数的机构补空 制度详情；初期空壳机构不补
    if (!机构.制度详情 && 机构.实有人数 > 0) {
      机构.制度详情 = {
        所属体系: '',
        机构性质: '常设机构',
        级别: '',
        职务体系: '',
        领导体制: '属地管理',
        上级业务主管: '无',
        决策机制: '一把手拍板',
        议事规则: '首长负责制',
        任命权限: _.get(variables, 'stat_data.组织结构.当前最高权力机构', '') + '任命',
        职级: '',
        编制数: 机构.实有人数 ?? 0,
        依托部门: '无',
        组长级别: '无',
        请示报告要求: '',
        保密等级: '内部',
      };
    }
  }
});
```

> 默认值均为安全的中性值，AI 在后续轮次按剧情修正。无需"已迁移"标志、无开关，每轮按需补空。

### 2.1 设计思路

三权指针、财政扩展、行政区划、人事、世界会议这五个顶层模块，初期不建立（混乱期/临时领导小组期无此需求）。**不采用 `Schema_扩展 = Schema.extend(...)` 平铺 extend，也不采用启用开关**，而是直接在主 `Schema` 中将这些字段标为 `.optional()`：

- 初期 AI 不填 → 字段缺省 → schema 校验通过 → 状态栏 `v-if` 不显示对应 Tab/徽标
- 建政后 AI 按剧情自然 `insert`/补值 → 字段出现 → 状态栏自然显示
- **无开关、无锁定、无 schema 切换、无数据迁移**
- 兜底规则 9-14 改为按数据存在性触发（如 `data.组织结构.当前监督机构` 存在且非 '无' 才执行规则 13），而非读 `扩展包已启用`

### 2.2 组织结构顶层：三权指针 + 财政扩展 + 行政区划 + 人事

直接在主 `组织结构Schema` 中追加 `.optional()` 字段（不破坏初期字段）：

```ts
const 组织结构Schema = z.object({
  // ...原有初期字段保持不变（当前体制/当前最高权力机构/机构/财政）

  // ── 决策-执行-监督三权制衡（建政后按需补）──
  当前决策核心: z.string().optional().describe('指向决策机构键名，通常=当前最高权力机构。建政后按需补'),
  当前执行首脑: z.string().optional().describe('指向行政执行机构键名，如 边区政府/人民委员会。尚未分立时不填'),
  当前监督机构: z.string().optional().describe('指向监察机构键名，如 监察委员会/纪委。尚未建立不填'),

  // ── 财政扩展（初期 财政 只有 收入来源/主要支出/财政状况，建政后追加以下子字段）──
  // 在 财政 子对象中追加 .optional() 字段：
  //   预算审批权: z.string().optional()
  //   审计机构: z.string().optional()
  //   财政备注: z.string().optional()

  // ── 行政区划（初期仅据点，控制区扩大后 AI 自然 insert）──
  行政区划: z.record(
    z.string().describe('区划名，如 当涂县/某乡/某村'),
    z.object({
      层级: z.string().describe('据点/村级/乡级/县级/地级/省级，可按剧情扩展'),
      治理机构: z.string().describe('指向 机构 record 中的键名'),
      人口: z.coerce.number(),
      治安状况: z.string().describe('稳固/稳定/紧张/动荡/失控'),
      与中央关系: z.string().describe('服从/观望/阳奉阴违/对抗'),
      备注: z.string(),
    }),
  ).optional().describe('控制区扩大后 AI 自然 insert 新区划；初期不填'),

  // ── 人事（干部管理，建政后按需补）──
  人事: z.object({
    干部管理原则: z.string().describe('如 "党管干部"/"军委任命"'),
    职级体系: z.string().describe('当前政权采用的职级体系描述'),
    关键岗位任命记录: z
      .array(z.object({
        岗位: z.string(),
        任命人: z.string(),
        任命主体: z.string(),
        任命时间: z.string(),
      }))
      .transform(data => _.takeRight(data, 10))
      .describe('近期关键岗位任命记录，保留最近 10 条'),
  }).optional().describe('建政后按需补；初期干部管理粗放，不填'),
});
```

### 2.3 世界模块：会议制度

直接在主 `世界Schema` 中追加 `.optional()` 字段：

```ts
const 世界Schema = z.object({
  // ...原有初期字段保持不变（当前日期/当前时间/政治阶段/当前大事件/穿越天数）
  当前会议: z.string().optional().describe('当前正在召开或即将召开的会议，常见取值：常委会会议/政治局会议/常务会议/部务会议/党组会议/民主生活会/扩大会议/领导小组会议/无。无会议时填 "无"；初期不填'),
});
```

### 2.4 非群友角色：可选补全字段

对长期 recurring 的重要 NPC（如反复出场的清廷官员、同盟会联络员），可按需在 `非群友角色Schema` 中追加 `履历` 和 `关键标签` 字段（格式同 `群友角色Schema`），标为 `.optional()`。一次性出场的 NPC 无需补全。

export function deriveAll_扩展<T extends RuntimeMvuData>(data: T) {

}

## 四、世界书扩展（变量更新规则.yaml 追加段落）

## 五、兜底规则 9-14 完整实现

在 `脚本/兜底校验/index.ts` 的 `VARIABLE_UPDATE_ENDED` 监听器中追加以下规则。**所有规则按数据存在性触发**（如 `机构.制度详情` 存在才执行规则 9/10/14，`组织结构.当前监督机构` 存在且非 '无' 才执行规则 13），不读任何启用开关。

### 规则 9：党政二元结构与双首长一致性

```ts
// 单层遍历 机构 record
// 机构.所属体系='党' → 一把手称呼应含 '书记'，决策机制应为常委会集体决策/民主集中制
//   若不匹配 → 校验警告（warn，不强制修改）
// 机构.所属体系='政' → 一把手为行政首长，决策机制通常为首长负责制
// 机构.所属体系='军' → 一把手=政委、二把手=军事主官；一把手 与 二把手 不可为同一人
//   → 若为同一人 → 校验警告 "双首长不可兼任"
// 机构.一把手 变更 → 机构.负责人 自动回填为 一把手
// 机构.一把手='空缺' 且 二把手≠'无' → 提示 "可由二把手代理"（不强制修改运作状态）
// 机构性质=议事协调机构 → 一把手通常=依托部门一把手兼任，校验一致性
// 党体系顶层机构一把手 与 政体系顶层机构一把手 为同一人 → 校验警告
//   （党领导政不等于党政一把手由同一人兼任，除非临时机构）
```

### 规则 10：条块关系一致性

```ts
// 单层遍历 机构 record
// 机构.领导体制='垂直管理' → 上级业务主管 必须非 '无'，否则校验警告
// 机构.领导体制='属地管理' → 上级业务主管 应为 '无'（纯块状）
// 机构.领导体制='双重领导' → 上级业务主管 必须非 '无'，且该机构须有非顶层的 上级机构
// 机构.领导体制='业务指导' → 上级业务主管 必须非 '无'
// 监察机构（运作状态为正常且类型含 '监察'/'纪委'）→ 领导体制 应为 '垂直管理'
```

### 规则 11：财政联动（财权事权匹配）

```ts
// 财政.财政状况='赤字' → 检查上一轮是否也为 '赤字'，若连续 3 轮
//   → 潜在危机追加「财政崩溃引发政权危机」
// 财政.财政状况='破产' → 事件队列追加「财政破产」
// 财政.审计机构='无' 且 穿越天数>60 → 潜在危机追加「长期无审计，腐败滋生」
// 财政.预算审批权 指向的机构键名不存在于 机构 record → 校验警告
// 遍历机构表：某机构 权势指数>80 → 检查 财政.主要支出 中是否有对应支出项
//   若无 → 提示 AI 补全该机构经费
```

### 规则 12：人事任命一致性

```ts
// 机构.一把手 或 二把手 变更（与 variables_before 对比）
//   → 人事.关键岗位任命记录 追加：
//     { 岗位: 机构名+职务, 任命人: 新任姓名, 任命主体: 机构.任命权限, 任命时间: 世界.当前日期 }
// 任命记录.任命主体 与 机构.任命权限 不一致 → 校验警告
// 机构.实有人数 > 编制数 → 提示 "超编，须说明原因或扩编"
// 机构.实有人数 < 编制数*0.5 且 运作状态='正常' → 校验警告 "人手不足却正常运作不合理"
```

### 规则 13：监督缺失 → 腐败/派系清洗危机

```ts
// 组织结构.当前监督机构='无' 且 穿越天数>90
//   → 潜在危机追加「无独立监督，派系清洗风险」
// 组织结构.当前监督机构 指向的机构 运作状态='瘫痪' 或 '半瘫痪'
//   → 潜在危机追加「监督机构失效，腐败蔓延」
// 监督机构.一把手 与 当前最高权力机构 指向的机构.一把手 为同一人
//   → 校验警告 "监督者与被监督者重合"
// 规则 8（派系斗争）触发且 当前监督机构='无' → 危机严重程度升一级
```

### 规则 14：领导小组组长级别校验

```ts
// 机构.机构性质='议事协调机构' → 组长级别 必须非 '无'，否则校验警告
// 机构.机构性质='议事协调机构' → 沿 上级机构 链找到其所协调的顶层常设机构
//   → 检查 组长级别 是否高于或等于该常设机构的 级别（或 职级）
//   → 若不够 → 校验警告 "领导小组组长级别不足以协调目标机构"
// 机构.机构性质='常设机构' → 组长级别 应为 '无'（常设机构无组长概念）
// 机构.机构性质='议事协调机构' 且 依托部门='无' → 校验警告
// 机构.机构性质='议事协调机构' 且 一把手='空缺' → 校验警告 "领导小组组长不可空缺"
```

> **校验级别说明**：
>
> - 所有规则 9-14 输出 **校验警告（warn）**，不强制回滚 AI 的修改。
> - 仅规则 2（死亡不可复活）和规则 4（核心设备损毁）为硬约束（阻止 AI 修改）。
> - 兜底脚本用 `console.warn` 记录警告，便于调试。
