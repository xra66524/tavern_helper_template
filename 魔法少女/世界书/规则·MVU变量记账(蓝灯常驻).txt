【铁律·必写命令】每轮结尾输出 <UpdateVariable>…</UpdateVariable>。<Analysis> 只是草稿不生效，真正改变量的只有 </Analysis> 后的命令行：改已有值用 _.set('路径',旧值,新值)；建新条目(招式/星器/行囊/委托/地点/重要人物/在场敌人)必须用 _.insert('集合','名字',{整条对象})——绝不能用 _.set 去建不存在的路径，否则框架直接丢弃、状态栏读不到。Analysis 里判 Y 的每项都要写出对应命令行(改值=set/建档=insert)，只写进 Analysis 括号里不算数。命令裸写别进代码块，中文路径别加"-"。

<变量更新规则>
你必须在每次回复的最末尾，依据【本轮正文中真实发生的事】，输出一段 <UpdateVariable> 块来更新上方变量。这是后台记账：正文不写、此处不记，状态栏就读不到，后续也无从追踪。务必与正文严格一致，不凭空回血/回蓝、不无中生有。

【输出前·先做简短分析】（用英文写在 <Analysis> 内，不解释理由）
- time passed: 本轮经历了多久 → 决定时间推进幅度
- list & judge: 逐一列出 stat_data 中相关变量，标记本轮是否需要更新(Y/N)
- threshold: 数值变量是否跨过阶段阈值(如侵蚀值满100进阶、经验满升级)

【格式·铁则】
1. 形式：_.set('变量路径', 旧值, 新值);//更新原因  —— 一条一行。
2. 路径按点号分层，与上方 stat_data 结构完全一致；不要带 stat_data. 前缀。
3. 旧值填当前快照里的值；新值为更新后的值；原因一句话点明触发事件。
4. 只输出本轮真正变化的变量；没变化的不要写。
5. 数值不越界：侵蚀值 0-100、生命/魔力不超过对应上限、好感按区间。
6. 字符串用引号包裹；除行尾 // 原因外，不要写别的注释。

【可用操作】
· 改已有值：       _.set('主角.状态.生命.当前', 100, 78);//被魔物爪击受创
· 新增集合条目(建档)：必须用 _.insert 一次性插入整条对象(不能用 set 去建不存在的路径) →
                   _.insert('重要人物', '林晚', {"性别":"女","身份":"异事局少尉","关系":"队友","好感":10,"等级":22,"生命":{"当前":180,"上限":180},"魔力":{"当前":150,"上限":150},"当前状态":"警戒待命","恶堕":"未染","解放":"未启","躯体":"利落短发","手段":"近身突进","钩子":"似在隐瞒旧案","最后出场":"上海·浦东分局 门口"});//建档
· 给已建条目改字段：_.set('重要人物.林晚.好感', 10, 18);//并肩作战后更信任
· 删除集合条目：   _.delete('在场敌人.腐蚀犬');//已被净化消灭
（集合 = 招式/星器/行囊/委托/地点/重要人物/在场敌人，均以「名字」为键）

【各集合的字段结构】（新增条目时按此填全，键名照抄）
· 招式[招式名]   ：类型(攻击/防御/辅助/领域…)、品阶(见习~传奇)、熟练度(生疏/熟练/精通…)、效果
· 星器[名称]     ：类型(剑/杖/弓/枪…)、品阶、契合(与主角裂痕契合度)、效果（注：星器即精神裂痕的锚定物，见魔法少女力量机制）
· 行囊[物品名]   ：类别(消耗品/材料/装备/任务/魔核…)、数量、备注
· 委托[委托名]   ：类型(讨伐/护送/调查/收集…)、状态(进行中/已完成/失败)、进展
· 地点[地点名]   ：所在区域(必须写成「城市·街区」如 上海·静安寺)、类型(学院/公寓/异事局/医院/商铺/车站/据点/野外…)
· 重要人物[姓名] ：性别、身份、关系(队友/对手/同学/引路人…)、好感(数值)、等级、生命{当前,上限}、魔力{当前,上限}、当前状态、恶堕(侵蚀阶段)、解放(淫堕解放阶段)、躯体、招式、手段、钩子(心事/秘密)、最后出场
· 在场敌人[名称] ：威胁(威胁等级)、生命{当前,上限}、手段、状态(完好/受伤/逃遁/被净化…)

【记账要点·一律照正文落实】
· 生命/魔力：受创扣生命；释放招式、维持变身按「占当前魔力上限的百分比」扣魔力(基础攻防≈0~3%，小技能5~12%，中技能15~25%，大招30~50%+)；休整回升，但一场恶战后不满血。
· 经验/升级：完成委托、净化魔物、突破历练给经验；当前经验≥升级所需 → 等级+1，按曲线刷新升级所需，跨档时阶级随之提升。
· 侵蚀：仅「精神世界接触」才涨侵蚀值(被魔人魔物侵染、久处瘴气、接受对面世界之力、与魔人缔约)；世俗之恶与战斗情绪一概不涨。本阶满100→进阶(未染→初蚀→沉沦→半堕→恶堕)，溢出带入新阶。淫堕/自我解放路径则跟踪解放度——解放度由自身意志主动推进，与精神世界接触无关，解放度满100→进阶(未启→羞启→坦然→盛放→极意)。侵蚀与解放互斥，同角色不同时推进两者。
· 时间/地点：行动消耗时间，推进 全局.当前时间；移动则更新 全局.地点 与 全局.去向；新到访的具名地点登记进「地点」表(所在区域写成 城市·街区)。
· 人物：新登场的要紧人物登记进「重要人物」；互动改 好感/关系/当前状态；离场更新 最后出场。
· 敌人：进入战斗的魔物/魔人登记进「在场敌人」；受创更新生命与状态；被消灭/逃离则 _.delete。
· 战服/躯体：恶堕推进（侵蚀进阶）或淫堕推进（解放进阶）或战斗破损时，更新 主角.状态.战服 及 主角.躯体 各部位外显。

【开局首条回复·一次性补全空缺】
游戏刚开始时(变量多为初始空值),除照档案填写外,务必主动把"已被开场剧情确立、或按设定理应存在、却仍空着"的字段一次性补全,宁可合理生成也不留空：
· 主角.外貌：档案没写就按人物气质自拟一个合理外貌。
· 全局.当前时间、全局.地点、全局.去向：按开场场景确立(如 '03月15日 08:00'、'上海·静安寺')。
· 主角.状态.处境/战况：按当前情景填(无战斗写'平静')。
· 主角初始招式：依其 系别/亲和 主动生成 1~2 个起手招式登记进「招式」(品阶与等级相称、熟练度多为'生疏/熟练')。哪怕开场没演出招式，也先建档。
· 主角星器：若设定中其有变身媒介/裂痕锚定物，生成 1 件登记进「星器」(类型/品阶/契合/效果)；确无则暂空。
· 主角行囊：按身份生成几样起步物品登记进「行囊」(如 通讯符、低阶魔核、应急药剂、少量灵币等)。
· 开场登场的人物：凡有名有姓或将持续互动者，登记进「重要人物」，等级/生命/魔力按其设定照战力标尺生成，并给 关系/好感 初值。
· 开场涉及的具名地点登记进「地点」(所在区域写成 城市·街区)；开场即战斗则把在场魔物/魔人登记进「在场敌人」。
此后按剧情增量更新即可。

【核心原则·主动建账不留空】
集合(招式/星器/行囊/委托/地点/重要人物/在场敌人)是"逐步生长"的：凡正文中已出现、或按情境理应存在的招式、人物、物品、地点、敌人，当轮即登记入对应集合并按字段表填全，不要等到被反复提及才补。空缺一旦被正文确立就当场补上、不再拖延。此点与旧版数据库一致——数据库会自动建行；这里由你用 _.insert('集合','名字',{整条对象}) 主动建项。铁律：建新条目只能用 _.insert，绝不能用 _.set 去建不存在的路径——对不存在路径的 set 会被框架直接丢弃，set 仅用于修改已存在的值。
