# 本地测试

## 一、快速开始

无需酒馆环境，直接双击 HTML 文件即可测试所有界面组件。

### 1. 构建

```bash
pnpm build:dev
```

### 2. 打开

直接打开以下文件：

```
dist/2039_ui/主面板/index.html
```

或者浏览器地址栏加参数：

- `file:///D:/tavern_helper_template/dist/2039_ui/主面板/index.html`
- `index.html?mock=1`
- `index.html#mock`

> 三种方式效果完全一样，页面会自动使用 `mockData.ts` 中的模拟数据。

---

## 二、三种 mock 触发模式

> `src/2039_ui/主面板/store.ts` 第 8 行 `isLocalMockMode()` 控制。

| 触发条件           | 说明                                   |
| ------------------ | -------------------------------------- |
| `file://` 协议     | 双击 `index.html` 直接打开时自动触发   |
| URL 包含 `?mock=1` | 手动加查询参数，适用于本地 HTTP 服务器 |
| URL 包含 `#mock`   | 手动加哈希，不会触发页面刷新           |

**非 mock 模式**（正常酒馆环境）：URL 非 `file://` 且无 mock 参数时，store 会通过 `getVariables()` 读取当前消息楼层的 `stat_data` 并每 2 秒轮询。

### 数据结构

`mockMvuData` 是一个完整的 `MvuData` 对象，结构对应 `src/2039_mvu/mvu_schema.ts` 中定义的 Schema：

```

├── 世界         ← 日期、时间、政治阶段、选举模式、当前大事件
├── 日本执政     ← 核心数据：首相、内阁、阁僚、事务次官会议、政治势力、政策
├── 日本国内情势 ← 通胀、治安、粮食、能源等民生指标
├── 国际势力动态
├── 舆论场
├── 各角色情况
├── 事件队列
├── $整体危机等级  ← 派生字段（直接写死，本地不走 deriveAll）
├── $外部压力指数
├── $执政合法性
└── $政府控制力
```

## 五、store.ts 本地/酒馆分流机制

```ts
// store.ts 核心逻辑
const isLocalMockMode = () => {
  const href = globalThis.location?.href ?? '';
  return href.startsWith('file:') || href.includes('mock=1') || href.includes('#mock');
};

const useMockDataStore = defineStore('mvu_data.2039_ui.mock', () => {
  const data = ref(mockMvuData as MvuData);  // 静态 mock 数据
  return { data };
});

const useRealDataStore = defineStore('mvu_data.message.2039_ui.main', () => {
  // 从酒馆 getVariables() + Schema.safeParse 读取，每 2 秒轮询
});

export const useDataStore = () =>
  isLocalMockMode() ? useMockDataStore() : useRealDataStore();
```

**关键点**：所有组件只调用 `useDataStore()`，不关心当前处于哪种模式。模式切换对组件透明。

## 六、index.html 的本地运行兼容层

正常酒馆环境中 Vue、Pinia、zod、lodash、`$` 均由酒馆全局注入。本地 `file://` 打开时这些全局变量不存在，因此在 `index.html` 中通过 `<script type="importmap">` + `<script type="module">` 手动加载：

```html
<script type="importmap">
  {
    "imports": {
      "vue": "https://testingcf.jsdelivr.net/npm/vue/+esm",
      "pinia": "https://testingcf.jsdelivr.net/npm/pinia/+esm",
      "zod": "https://testingcf.jsdelivr.net/npm/zod/+esm",
      "lodash": "https://testingcf.jsdelivr.net/npm/lodash/+esm"
    }
  }
</script>
```

同时提供了一个最小 `$` 兼容实现（模拟 jQuery ready），Vue 应用入口 `index.ts` 中的 `$(() => { createApp(App).mount('#app'); })` 才能正常执行。

> **注意**：这些 `<script>` 标签最终会被 webpack 打包进 `dist/index.html`，因此修改 `src/.../index.html` 后要重新构建。

## 七、常见问题

### Q: 打开 index.html 白屏，控制台报错 "Vue is not defined"

**原因**：直接用旧版 dist 产物，没有 vue/pinia/zod/_ 全局变量。

**解决**：重新 `pnpm build:dev`。

### Q: 修改 mockData.ts 后刷新没变化

**原因**：修改源码后需要重新构建。

**解决**：运行 `pnpm build:dev` 或开启 `pnpm watch` 自动构建。

### Q: 想测试某个边界情况（比如首相空位）

**解决**：修改 `mockData.ts` 中 `日本执政.首相状态` 为 `'空位'`，`首相姓名` 设为 `''`，重新构建。

### Q: 在酒馆中会不会误用 mock 数据

**不会**。酒馆中打开的 URL 是 `http://` 或 `https://` 且不含 `mock=1`/`#mock`，`isLocalMockMode()` 返回 `false`，store 走真实数据路径。

### Q: 本地 HTTP 服务器开发时也想用 mock

在 URL 后面加 `?mock=1` 即可，例如：

```
http://localhost:8080/dist/2039_ui/主面板/index.html?mock=1
```
